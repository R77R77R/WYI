\begin{abstract}
The WYI project is a multi-modal AI-driven platform [cite: 3] designed to automate the recognition, classification, and validation of financial documents[cite: 3]. This manual serves as a comprehensive guide to the system architecture, component management, and AI integration protocols[cite: 4].

\chapter{Project Introduction}
\section{Objective}
The primary goal of WYI is to minimize manual data entry in bill management[cite: 6]. By utilizing the Google Gemini 1.5 Flash model, the system achieves high-accuracy parsing of complex document layouts[cite: 6].

\section{System Requirements}
\begin{itemize}
    \item \textbf{Runtime:} .NET 9.0 Core with F\# [cite: 7]
    \item \textbf{Database:} SQL Server / PostgreSQL [cite: 8]
    \item \textbf{Authentication:} Clerk.js Integration [cite: 12]
    \item \textbf{AI Engine:} Google Gemini Multimodal API [cite: 6, 10]
\end{itemize}

\chapter{System Architecture}
\section{Backend: UtilKestrel}
The backend handles critical logic including file orchestration, secure data persistence, and AI prompt engineering[cite: 7, 8]. It leverages F\# for robust asynchronous processing[cite: 7, 10].

\section{Frontend: Vue 3 Ecosystem}
The frontend manages user interactions and multi-file uploads via the \texttt{UploadBills} module[cite: 9]. State is managed through a reactive store to ensure consistency between the AI analysis and the UI[cite: 9].

\chapter{Technical Implementation}
\section{Gemini Multimodal Analysis}
The following implementation demonstrates the asynchronous task for processing document images through the Gemini API[cite: 10, 11].

\begin{lstlisting}[language=ml, caption=F\# Gemini Integration Task]
// GeminiMultimodal Task Implementation
let GeminiMultimodal output apiKey model prompt files = task {
    use client = new HttpClient()
    // Extended timeout to handle multi-image processing
    client.Timeout <- TimeSpan.FromMinutes(5.0) 
    
    // API endpoint construction and POST execution
    let! response = client.PostAsync(url, content)
    return! response.Content.ReadAsStringAsync()
}
\end{lstlisting}

\chapter{Troubleshooting}
\section{Network and Authentication}
If a \texttt{NetworkError} occurs during Clerk.js authentication, ensure \texttt{clerk.accounts.dev} is accessible[cite: 12]. In restricted regions, using a VPN with the Lightway protocol (UDP/TCP) is recommended to maintain a stable session[cite: 13].

\section{Timeout Issues}
A \texttt{TaskCanceledException} typically indicates that the AI analysis exceeded the default 100-second window[cite: 14]. The system is configured to use a 300-second timeout to accommodate high-resolution uploads[cite: 14].
